Le projet en bref

Aucun foyer ivoirien ordinaire n'a de compteur intelligent. EcoWatt en fabrique un, accessible : des prises qui mesurent chaque appareil, et une IA qui coupe le gaspillage et décale les usages hors des heures de pointe.

Le problème

  • Les factures montent sans qu'on comprenne pourquoi.
  • Zéro visibilité sur la consommation, appareil par appareil.
  • Les pics en heure de pointe surchargent le réseau et provoquent des coupures.
  • L'électricité venant surtout du gaz, chaque kWh gaspillé émet du CO2.

La solution

Chaque prise intelligente (ESP32 + capteur + relais) mesure la consommation réelle et peut couper l'appareil. Une IA reçoit les mesures, décide quoi couper ou décaler, commande la bonne prise, et explique chaque décision.

Comment l'IA décide (4 facteurs)

FacteurRôle
Priorité de l'appareilEssentiel (jamais coupé), reportable (décalable), confort.
Le momentEn heure de pointe, le réseau est chargé et l'électricité la plus polluante.
L'objectifÀ l'approche d'un budget, on déleste les appareils reportables.
La conso réelleMesurée par la prise, en temps réel.

Comment utiliser l'application

Lancer l'application (en local)

# Backend (API + IA)
cd backend
.venv/Scripts/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

# Frontend (dashboard)
cd app
npm run dev        # ouvre http://localhost:5173

Naviguer

La barre latérale gauche donne accès à toutes les pages : Tableau de bord, Assistant IA, Appareils, Décisions IA, Impact, Historique, Réseau, Réglages. Sur mobile, un bouton menu ouvre la barre en tiroir.

Mode démo ou réel

  • Mode démo (par défaut) : l'app rejoue un scénario simulé, sans matériel. Idéal pour présenter.
  • Mode réel : clique la carte "Mode démo" en bas de la barre pour basculer. Le dashboard se connecte au backend et affiche les vraies prises. Vide tant qu'aucun matériel n'est branché.

Parler à l'IA

  1. Va sur Assistant IA.
  2. Écris une question ("Quel appareil consomme le plus ?") ou un ordre ("Coupe la bouilloire").
  3. Ou clique le micro (Chrome/Edge), parle, et l'IA n'envoie qu'à la fin de ta phrase.
  4. Active Lecture vocale pour qu'elle lise ses réponses à voix haute.

Mode sombre

Le bouton soleil/lune en haut à droite bascule en noir pur. Le choix est mémorisé.

Brancher le matériel — WiFi

Le boîtier et le PC (backend) doivent être sur le même réseau WiFi. Le boîtier vise l'IP locale du PC. Dès sa première mesure, l'appareil apparaît tout seul.

a. Matériel par prise

  • ESP32 (le cerveau du boîtier).
  • Capteur de courant (ex. SCT-013 + circuit de mesure) : mesure la consommation.
  • Module relais conçu pour le secteur : coupe/rallume l'appareil.
Sécurité 220V
On manipule du 220 volts, c'est dangereux. Modules secteur uniquement, jamais de fil nu sous tension, montage supervisé par quelqu'un qui s'y connaît.

b. Côté PC

  1. PC et ESP32 sur le même WiFi.
  2. Récupérer l'IP locale du PC (ipconfig → adresse IPv4 du Wi-Fi). Exemple actuel : 192.168.1.19.
  3. Lancer le backend en écoute réseau : uvicorn app.main:app --host 0.0.0.0 --port 8000.
  4. Autoriser Python dans le pare-feu Windows (fenêtre au 1er lancement).

c. Configurer le firmware

Dans firmware/src/main.cpp, en haut :

WIFI_SSID        = "TON_WIFI";          // même WiFi que le PC
WIFI_PASS        = "TON_MOT_DE_PASSE";
BACKEND_HOST     = "192.168.1.19";      // IP locale du PC
BACKEND_PORT     = 8000;
PRISE_ID         = "p1";                // unique par prise
DEVICE_ID        = "kettle-1";
DEVICE_NAME      = "Bouilloire";        // nom affiché
DEVICE_PRIORITE  = "reportable";        // essentiel | reportable | confort

d. Flasher l'ESP32

cd firmware
pio run -t upload      # compile et téléverse (USB)
pio device monitor     # logs : [wifi] IP ... puis [ws] connecte au backend

e. Voir l'appareil apparaître

  1. Dans l'app, passer en Mode réel.
  2. Alimenter le boîtier.
  3. À la première mesure, l'appareil apparaît automatiquement sur le dashboard, la conso monte, l'IA peut le piloter.
Calibration (important)
Dans readWatts(), la constante AMPS_PER_ADC est à 0 par défaut. Pour des watts justes, branche une charge connue (ex. ampoule 60 W), relève la valeur du capteur, et ajuste ce facteur. Sans calibration, la prise remonte 0 W (mais connexion et relais marchent).

Brancher le matériel — Bluetooth (option)

Le WiFi reste recommandé (portée, always-on, multi-prises). Le Bluetooth est un plan B quand il n'y a pas de WiFi. Le cerveau (IA, impact, dashboard) est identique.

Côté PC

pip install bleak
set ECOWATT_BLE=1
.venv/Scripts/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

Côté boîtier

  • Flasher firmware/ble/main_ble.cpp à la place du firmware WiFi.
  • Mêmes réglages PRISE_ID / DEVICE_ID / DEVICE_NAME / DEVICE_PRIORITE.

Le boîtier annonce un nom EcoWatt-… ; le backend le détecte, s'abonne aux mesures et envoie les ordres. WiFi et Bluetooth peuvent coexister sur le même dashboard. Nécessite un adaptateur Bluetooth sur le PC.
